Global Variables Example

Shows how the global variables that are available in the Sling HTL Engine are handled in Storybook AEM.

properties

the properties (aka ValueMap) of the current resource:

Example:

${"
\u0024{properties.text\u007d
" @ context = "text"}

gives:

${properties.text}

log

Maps to console. only useful in use classes.

wcmmode

Always {}

component

Storybook component object.

Unsupported

The following global variables are all mapped to a dummy object that return a debug message.

componentContext, currentDesign, currentNode, currentPage, currentSession, currentStyle, designer, editContext, inheritedPageProperties, out, pageManager, pageProperties, reader, request, resolver, resource, resourceDesign, resourcePage, response, sling, slyWcmHelper, xssAPI,

Example:

${"
\u0024{currentNode.path\u007d
" @ context = "text"}

gives:

${currentNode.path}
